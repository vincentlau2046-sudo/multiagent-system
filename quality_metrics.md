# 量化质量指标与置信度计算公式

## 1. 质量评估维度

### 完整性 (Completeness) - 权重: 30%
- **定义**: 任务要求的所有关键要素是否都已覆盖
- **评分标准**: 
  - 100%: 所有要求要素完整覆盖
  - 80%: 覆盖主要要素，次要要素有少量遗漏  
  - 60%: 覆盖核心要素，但有明显遗漏
  - 40%: 仅覆盖部分要素
  - 20%: 要素覆盖严重不足
  - 0%: 完全偏离任务要求

### 准确性 (Accuracy) - 权重: 35%
- **定义**: 提供的信息、数据、分析是否准确无误
- **评分标准**:
  - 100%: 所有信息经过验证，完全准确
  - 85%: 关键信息准确，细节有轻微误差
  - 70%: 主要信息基本准确，存在可接受的误差范围
  - 50%: 部分信息存在明显错误
  - 25%: 关键信息错误较多
  - 0%: 信息完全错误或虚构

### 深度 (Depth) - 权重: 20%
- **定义**: 分析的深度、洞察力和专业性
- **评分标准**:
  - 100%: 提供深度洞察、独到见解和专业分析
  - 80%: 分析较为深入，有一定专业性
  - 60%: 基础分析到位，但缺乏深度
  - 40%: 分析较为表面
  - 20%: 仅有简单描述，无实质分析
  - 0%: 无任何分析内容

### 实用性 (Practicality) - 权重: 15%
- **定义**: 结果是否具有实际应用价值和可操作性
- **评分标准**:
  - 100%: 提供具体可执行的建议和方案
  - 80%: 建议具有较强实用性
  - 60%: 建议基本可用，需要进一步细化
  - 40%: 建议较为抽象，实用性有限
  - 20%: 建议缺乏实际价值
  - 0%: 完全不具实用性

## 2. 置信度计算公式

### 基础置信度计算
```
置信度 = (完整性得分 × 0.3) + (准确性得分 × 0.35) + (深度得分 × 0.2) + (实用性得分 × 0.15)
```

### 任务类型调整系数
- **intelligence_collection** (情报收集): 准确性权重提升至45%，深度权重降至10%
- **technical_analysis** (技术分析): 深度权重提升至30%，实用性权重降至10%  
- **content_creation** (内容创作): 实用性权重提升至25%，深度权重保持20%
- **complex_research** (深度研究): 各维度权重均衡，深度权重提升至25%

### 最终置信度公式
```
最终置信度 = 基础置信度 × 任务类型调整系数
```

## 3. 审核决策阈值

- **≥90**: 自动通过，无需主审
- **70-89**: 需要主审确认
- **<70**: 审核失败，触发React机制

## 4. 特殊情况处理

### 时间敏感任务
- 允许降低深度和实用性要求，但准确性不得低于80%

### 教育类任务 (math_tutoring)
- 准确性权重提升至50%，其他维度相应调整
- 置信度阈值提高至95%

### 创意类任务 (content_creation)
- 允许在准确性上适度放宽（不低于70%），但实用性和完整性要求提高
